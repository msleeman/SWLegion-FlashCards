# SWLegion FlashCards

A web-based flashcard app for learning all Star Wars: Legion keywords. Browse the catalog, study with flip-cards, track what you've learned, and take notes.

---

## Quick Start

```bash
# Full rebuild (scrapes web + PDF, downloads images, writes cache)
py -m src.build

# Fast rebuild from cache (no network, no scraping)
py rebuild_html_only.py

# Re-scrape only garbage/missing definitions
py refresh_definitions.py

# Open the result
start dist/index.html
```

> Legacy: `py build_swlegion_v4.py` still works (thin shim to `src.build`).

## THE RULE: Never edit generated files

| File | Status | Edit this instead |
|---|---|---|
| `dist/index.html` | Generated | `template/index.html`, `template/app.css`, `template/app.js` |
| `dist/images/` | Generated | `overrides/<stem>.<ext>` |
| `cache/card_data.json` | Generated + gitignored | `overrides/` or `src/` |

---

## Override System

Drop files in `overrides/` named after the keyword stem. Overrides always win over scraped/cached data.

### Stem rules
- `Backup[]` → `backup`
- `Pierce X` → `pierce`
- `Coordinate: Type/Name[]` → `coordinate`
- `Sharpshooter 1` → `sharpshooter`

### What you can override

| File | Effect |
|---|---|
| `overrides/<stem>.md` | Full rules text |
| `overrides/<stem>.summary.md` | One-liner shown in catalog |
| `overrides/<stem>.png\|.webp\|.jpg` | Card image |
| `overrides/<stem>.txt` | Image credit line |

### Survival guarantee

| Where the edit lives | Survives `rebuild_html_only.py`? | Survives `py -m src.build`? |
|---|---|---|
| `overrides/<stem>.md` | ✅ Yes | ✅ Yes |
| `overrides/<stem>.summary.md` | ✅ Yes | ✅ Yes |
| `overrides/<stem>.<img>` | ✅ Yes | ✅ Yes |
| In-browser edit (localStorage) | ✅ Yes (browser only) | ✅ Yes |
| `cache/card_data.json` (hand-edited) | ❌ Overwritten | ❌ Overwritten |

---

## Image Priority Chain

1. `overrides/<stem>.png|webp|jpg` — your image wins
2. Official unit card from LegionHQ2 CDN
3. Wikimedia Commons fallback

## Versioning

Version is derived from `git describe --tags --always --dirty` — no version file to conflict across PCs. The current version shows in the app footer.

To release a new version: `git tag v5.x.y && git push origin v5.x.y`

## Multi-PC Workflow

```bash
git pull --rebase        # always start here
# ... make changes ...
py -m src.build          # or py rebuild_html_only.py for fast rebuild
git add -A
git commit -m "..."
git push
```

GitHub is the source of truth. Friends clone the repo and open `dist/index.html`.

## Full Build vs Fast Rebuild

| Script | Network? | Scraping? | PDF? | Use when |
|---|---|---|---|---|
| `py -m src.build` | Yes | Yes | Yes | First run, new keywords, fresh machine |
| `py rebuild_html_only.py` | Only if image missing | No | Yes | CSS/JS tweaks, override edits |
| `py refresh_definitions.py` | Yes | Definitions only | No | Fix bad definitions without full rebuild |
